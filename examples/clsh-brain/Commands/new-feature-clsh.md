Plan and implement a new feature for clsh.dev: $ARGUMENTS

1. Read the [[Execution-Plan]] and [[Features]] to understand current scope
2. Read the technical plan in `Docs/compass_artifact_*.md` for relevant implementation details
3. Check if the feature already exists in the roadmap or is related to existing features
4. Create a feature spec using the `Templates/feature-spec.md` template
5. Output an implementation plan:
   - **Feature**: What it does and why users need it
   - **Priority**: P0 / P1 / P2
   - **Agent**: Which agent persona should implement this
   - **Files to create/modify**: Exact paths
   - **Implementation approach**: Step-by-step
   - **Testing plan**: How to verify it works
   - **Acceptance criteria**: When is it done?
6. Update [[Features]] with the new feature entry
7. If approved, implement the feature
